    # Vue-js Learing

    官方简介：

    Vue (读音 /vjuː/，类似于 view) 是一套用于构建用户界面的渐进式框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动

    # Hello Vue

    • vue采用的声明式编程。

      • 类型：string | HTMLElement

      • 类型：Object | Function（组件当中data必须是一个函数）
      • 作用：Vue实例对应的数据对象

    # v-for列表展示

    • v-for后面给数组追加元素的时候，新的元素也可以在界面中渲染出来
    <body>
    <div id="app">
        <li v-for="item in number">{{item}}</li>
<!--        v-for是vue指令的一种，往后会详细提及，作用是自动创建项目-->
    </div>
    <script src="vue.js"></script>

        el:'#app',
        data:{
            number:['1','2','3']//数组列表
        }
    })
</script>
</body>

    • 列表的数据是响应式的，可以在控制台进行添加。app.number.push('4');页面中就会自动创建一个新的并显示4。

    • 案例：计数器：

      <body>
    <div id="app">
       <h2>当前记数：{{counter}}</h2>
        <button @click="add">+</button>
        <button @click="sub">-</button>
    </div>
    <script src="vue.js"></script>
    <!--        v-for是vue指令的一种，往后会详细提及，作用是自动创建项目-->
<script>
   const app = new Vue({
        el:'#app',
        data:{
            counter:0
        },
       methods:{
            add:function (){
                this.counter++
            },
           sub:function () {
               this.counter--;
           }
       }
    })
</script>
</body>

    • methods属性：用于在Vue对象定义方法

      • 类型：{[key:string]:Function}
      • 作用：定义Vue的一些方法，可以在其他地方调用，也可以在指令中使用。

    • @click，该指令用于监听某元素的点击事件，并且需要指定当发生时，执行的方法，该方法一般是methods中定义的方法。

    # 计算属性computed

    • vue固定属性

    • 计算属性里面写函数，调用函数不用括号，因为计算属性里面定义的内容类似于一种属性，可以直接用。

      <body>   
    <div id="app">  
        <h2>{{fullname}}</h2>  
    </div>      
    <script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>    <script>     
    const app = new Vue({     
        el:'#app',   
        data:{     
            firstname:'Lebron', 
              lastname:'James'      
        },         
        computed:{      
            fullname:function (){       
                return this.firstname+' '+this.lastname;   
            }           
        },       
    })    </script></body>

    • 计算属性一般用来计算某些值,例如计算总价格

      <body>   
    <div id="app">       
        <h2>总价格：{{Totalprice}}</h2>     
    </div>      
    <script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>    <script>      
    const app = new Vue({      
        el:'#app',          
        data:{              
            books:[              
                {id:110, name:'Unix编程艺术',price:119}, 
                {id:111,name:'代码大全',price:105},    
                {id:110, name:'深入理解计算机原理',price:89}, 
                {id:110, name:'现代操作系统',price:87},      
            ]          
        },          
        computed:{          
            Totalprice:function (){  
                let result=0;        
                for(let i=0;i<this.books.length;i++){ 
                    result+=this.books[i].price;   
                }            return result;      
            }        
        }     
    })    
    </script></body>

    # 计算属性的缓存（与methods的区别）

    <body>    <div id="app">   
    <h2>{{getFullname()}} </h2> 
    <h2>{{getFullname()}}</h2>   
    <h2>{{Fullname}}</h2>     
    <h2>{{Fullname}}</h2>    
    <h2>{{Fullname}}</h2>    
    <h2>{{Fullname}}</h2>  
    </div>     
    <script src="https://cdn.jsdelivr.net/npm/vue@2/dist/vue.js"></script>    <script>     
    const app = new Vue({        
        el:'#app',          
        data:{                
            firstname:'Kobe', 
              lastname:'Bryant'          
        },    
        methods:{           
            getFullname:function (){  
                console.log( this.firstname+' '+this.lastname);
                return  this.firstname+' '+this.lastname;  
            }       
        },          
        computed:{       
            Fullname:function (){        
                console.log( this.firstname+' '+this.lastname) 
                return this.firstname+' '+this.lastname;      
            }         
        }       
    })   
    </script></body>
    • 以上代码可以解释为什么computed比methods要快，在控制台里，调用getFullname（）的methods方法，执行两次就在控制台打印了两次，而通过Fullname属性的computed方法，只在控制台输出了一次，说明就运行了一次。

            </form>
        </div>     
    var app = new Vue({  
        el:'.app',   
                methods:{    
                    onSubmit(){    
                        console.log("onSubmited");      
                    },   
                            myclick(){
                                //submit       
                            }   
                }
    });

    • v-for使用过程添加key
    • 当向控制台输入在B和C间插入一个F的命令时，vue通过Diff算法把下面的C改成了F，再添加一插入一个C，这样的做法效率很低。
    • 要是使用key给每一个节点都做一个唯一标识
      • Diff算法就可以正确的识别此节点
      • 找到位置直接插入新的节点

    # 具名插槽

    • 我们在开发网页时同一个子组件可以通过插槽来修改某些部位变成不同的样子，但是一个子组件不可能只用一次，所以仅插入一个插槽不能解决问题，但是引入多个插槽，会出现同名问题。 例如：我只想插入中间插槽，但是其他两个插槽也同时改变了。
    <body>
    <div id="app">
           <cpn><span>lbwnb</span></cpn>
        </div>
    <template id="cpn">
       <div>
          <slot >左边</slot>
           <slot >中间</slot>
           <slot >右边</slot>
       </div>
    </template>
    • 解决方法很简单，只要在插槽上标注名字即可。
    <div id="app">
      <cpn><span slot="center">lbwnb</span></cpn>
    </div>
<template id="cpn">
   <div>
      <slot name="left">左边</slot>
       <slot name="center">中间</slot>
       <slot name="right">右边</slot>
   </div>
</template>

    # ES6的模块化

    • module
      • 引入js文件时注入 type="module",使文件模块化，变量函数可复用，不互相干扰
    <script src="aaa.js" type="module"></script>
    • export
      • 导出，将变量对象函数导出，使他们可以被引入
    var name='小明';
var age = 18;
var flag = true;
function sum (a,b){
    return a+b;
}
export {
    flag,sum//这里只有flag和sum函数暴露了
	//或者在声明变量时加expoet。  export var name	；
}
    • import
      • 导入，将可以导入的变量引入
    import {
    flag,sum
} from "./aaa.js";//这里不可以简写
    • export default
      • 导入者可以命名变量，但只能传一个变量
    //aaa.js
export default function address(){
    console.log("湖南");
}
//bbb.js
addr();//湖南 
import addr from "/aaa.js"
